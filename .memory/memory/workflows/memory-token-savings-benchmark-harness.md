The Memory token-savings benchmark harness supports `validate`, `precheck`, `run`, and `report` under `pnpm bench:memory -- ...`. Ready tasks include the basic `p-limit-map-settled` pilot and the complex tasks `p-limit-priority-queue`, `zod-treeify-options`, and `vite-fs-access-diagnostics`; generated artifacts live under ignored `.bench/memory-token-savings/` as `results.json`, `results.csv`, `report.md`, per-run summaries, logs, diffs, verification results, and curation caches. The harness supports `--memory-mode bootstrap|curated` and `--run-label <label>`; curated mode creates one reusable Memory cache per repo/run-label under `.bench/memory-token-savings/curated/<run-label>/`, runs `memory setup --no-view`, uses a separate high-reasoning curator Codex pass to save reusable repo Memory without benchmark-task leakage, validates with `memory check` and `memory load`, then reuses that curated Memory for Memory-condition jobs. The current curated ready-subset command is `pnpm bench:memory -- run --full --allow-partial --repeats 1 --memory-mode curated --run-label curated-v1`.
