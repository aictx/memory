Memory pivoted from a per-task wiki (load before every task, remember after) to a product graph built once via agent-driven indexing, queried on demand, and reconciled by diff-driven sync. Reasons: the May 2026 token-savings benchmark showed per-task loading costs tokens (+21.7% net curated) instead of saving them; the agent-facing surface (27 verbs, 12 kinds, 22 facets) was too large for reliable agent use; and the underserved layer is product knowledge (intent, stage, rationale) that agents cannot re-derive from code. Reference point: graphify validated index-once/query-on-demand for the code layer; Memory owns the product layer. Schema broke from v4 to v5 with no migration.
